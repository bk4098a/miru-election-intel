Map Engine
세계 지도 — TopoJSON Choropleth
D3.js 미사용. TopoJSON client + 순수 Canvas API로 직접 구현. Natural Earth 투영법(Natural Earth projection) 수식을 JS로 직접 계산.
// Natural Earth projection — 핵심 상수
const NE_XMAX = 2.7353; // π × 0.8707
const NE_YMAX = 1.3401; // lat=85°에서의 y값 (이전 0.85는 오류 → 북부 국가 잘림)

// 렌더링 흐름
1. WORLD_ATLAS (인라인 JSON) → topojson.feature() → GeoJSON features
2. features 순회 → numeric id → ISO3_NUM 역방향 조회 → iso3
3. iso3가 COUNTRIES/COUNTRIES_BIO에 있으면 → 색상 결정 (기계투표/생체인식/Miru)
4. feature.arcs → 폴리곤 좌표 → project(lon, lat) → Canvas 경로
5. WORLD_DOTS (40개국 좌표) → 점·레이블 오버레이
6. 마우스 hover → 툴팁 표시 (국가명·공급사·기기·포털)
NE_YMAX = 0.85 버그: lat=60° (러시아·캐나다·북유럽)의 y값이 이미 0.85를 넘어서 캔버스 밖으로 나가 북부 국가들이 잘렸음. 정확한 값 1.3401로 수정 후 해결.

Classification

예: 케냐 KIEMS — 지문인식 후 종이 투표용지 교부, 수작업 개표.
Category C (B와 겹침)
결과전송 5개국
수개표 후 집계 결과만 전자 전송.
투표·개표 자체는 수작업.

예: 나이지리아 IREV, 케냐 Form 34A 사진 전송.
이 분류가 중요한 이유: 케냐·남아공 같은 국가는 선거 현장에 전자기기가 보이기 때문에 "기계투표 국가"로 오해하기 쉽다. 그러나 투표·개표는 수작업이므로 Miru OMR 납품 대상이 아니다. 대신 생체인식 기기·결과전송 솔루션 납품 기회국으로 분류한다.

포털 링크는 status에 따라 <a> (접근 가능) 또는 <span> (불가) 선택.
5
패널 HTML 생성 (7개)
region_panel("apac/europe/americas/mena") 4개 + bio_panel() + about_panel()

각 패널은 <div id="panel-{id}" class="panel"> 구조.
6

CSS는 하나의 f-string. 브랜드 색상은 Python 딕셔너리 B에서 직접 주입.
f-string 안 중괄호 규칙: CSS/JS 리터럴 중괄호는 {{/}} 이중 이스케이프 필수.
Chart.js·TopoJSON·WorldAtlas 데이터 모두 인라인 삽입 → 단일 파일.
7
파일 저장
data/election_intel_report.html에 UTF-8로 저장. 816KB 단일 파일.
UI Logic
7개 패널 탭 전환 + KPI 팝업 구조
/* 탭 전환 — CSS + JS 이중 제어 */
.panel { display: none; }
.panel.active { display: block; }

function showPanel(panelId) {
querySelectorAll('.panel').forEach(p => p.style.display = 'none');
getElementById('panel-' + panelId).style.display = 'block';
}

/* KPI 팝업 — 클릭 시 포털 링크 목록 표시 */
KPI_PORTAL_JSON = { miru, mv, bio, all, portals } ← Python에서 JSON으로 주입

kpiEl.addEventListener('click', () => {
const data = KPI_PORTAL_JSON[kpiEl.dataset.kpi];
popup.innerHTML = data.items.map(item => {
// item.url 있으면 → 새 탭에서 포털 열기
// item.panel 있으면 → 해당 패널로 이동
});
popup.style.display = 'block';
});
CSS + JS 이중 제어를 쓰는 이유: CSS display:none만 쓰면 특이성(specificity) 충돌로 일부 브라우저에서 패널이 동시에 보이는 버그 발생. style.display 인라인 스타일은 모든 CSS 규칙보다 우선순위가 높아 확실하게 동작.
